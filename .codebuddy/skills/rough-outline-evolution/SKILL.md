# 粗纲穷举进化 v6.1

独立的叙事 Ralph Loop 协议，专为粗纲（卷/事件级）叙事结构设计的 7 轮角色轮换 + 批量交互。
**致命反模式**: R4+ 凭记忆输出 `found:0` 而不做实际审查 → 假收敛。
**修复**: 每轮必须产出不可伪装的 3 项证据。

**粗纲 vs 细纲**: 粗纲查"这个故事能成立吗？"（主线/战力/伏笔/角色弧）。细纲查"这一章读者会翻页吗？"（钩子/节奏/爽点/衔接），由 `chapter-evolution` Skill 负责。

## 术语定义

| 缩写 | 含义 | 说明 |
|:---:|------|------|
| C | Critical | 根本性错误，交付物不可用 → 必须立即修复 |
| H | High | 重大缺陷 → 本轮必须修复 |
| M | Major | 违反需求或最佳实践 → 交付前必须修复 |
| L | Low | 轻微改进 → 可选 |
| zero | 零发现计数器 | C+H+M=0时+1；>0时归零。仅R8时检查（R1-R7逐轮执行，无视zero值） |
| N | 当前轮次 | R1-R8，达到R8为截断 |

## 前提条件

执行前必须确认以下输入存在且非空：
- `outline/rough_outline.md` — 粗纲主文件（必须有实质内容，不能仅为占位符）
- `outline/summary.md` — 故事概要（可选但建议存在）
- `canon/world.md` — 世界观设定（可选但建议存在）
- `canon/characters.md` — 角色设定（可选但建议存在）

**空输入处理**: 若粗纲为空，Skill 报 [BLOCK] 并提示"请先生成粗纲内容（S0阶段），再执行进化审查"。

## 每轮输出格式 (MANDATORY)

```
[Round N] role:角色名

5项检查单:
① [标准1] → ✅/❌ (依据: 粗纲段落/事件 — 具体引用)
② [标准2] → ✅/❌ (依据: ...)
...
外部引用: "[原句]" — 来源: [名称], [年份]
取样证据:
  - 粗纲:章节/事件 — "[原文]"
  ...
发现: C:X/H:Y/M:Z/L:W → fixed:X+Y+Z | zero:N → continue/converge
```

**没有 role/source/samples 三项 = 本轮无效, 不计数**

## 每轮必须产出的证据（缺一即本轮无效）

``` 
① 5项检查单: R1-R2可用定性描述(每项≥2句具体判断,含引用); R3-R7必须 ✅ 或 ❌ 并附依据
② 外部来源: >=2次WebSearch + >=1次WebFetch，至少1条原句(WebFetch失败时降级为WebSearch snippet，标 [降级引用])
③ 取样证据: 从粗纲中引用至少 3 处具体事件/台词/章节号作为检查依据
```

## 8 轮角色轮换

**角色定义、人格注入、5项检查单、方法论 → `roles/` 目录**（每轮审查前按需加载当前角色文件）

角色轮换顺序: 白金作者直觉→情感真实性→逻辑审查→世界观内洽→序列因果→角色一致性→伏笔结构→横向对比

## R1-R2 定性评分参考 (防止主观漂移)

R1-R2 不要求 ✅/❌ 二元判断，但**每项必须包含 2 个要素**，缺失 = 无效：

| 要素 | 要求 |
|:---:|------|
| **具体引用** | 从粗纲中引用 1 处原文/事件/台词 |
| **判断理由** | ≥1 句解释为什么这样判断 |

**禁止**: 只说"我觉得会"/"应该会" 而无引用+理由。

## 执行模式：批量交互（默认）

采用 3 批暂停机制：每批跑完暂停，作者审阅+提意见，AI 修完进下一批。

| 批次 | 轮次 | 角色 | 层 | 暂停时机 |
|:---:|:---:|------|:---:|------|
| 第1批 | R1-R2 | 直觉→情感 | 感知层 | R2 输出后暂停 |
| 第2批 | R3-R7 | 逻辑→世界观→因果→角色→伏笔 | 结构层 | R7 输出后暂停 |
| 第3批 | R8 | 横向对比 | 市场层 | R8 输出后出收敛报告 |

### 作者反馈协议

每批暂停后，AI 在当前对话中展示本批汇总（列出所有发现），等待作者反馈。作者反馈方式：
- 直接在对话中回复，逐条确认/否决/修改方案
- 示例: "R1的开篇问题，我同意改为XX版本。R2的情感问题先不改，进下一批再看"
- 反馈不要求特定格式，AI 从对话中提取决策并更新修正确认清单

## 执行流程

```
0. ROLE-LOAD: 读取 roles/r{N}-{name}.md
1. MANIFEST: 检查修正确认清单 — 跳过已确认修复的问题域
1.5 SAMPLE-DB: 读取 data/analysis/末世/synthesis/rhythm_benchmark.md（genre_synthesizer 自动生成），检查生成时间戳。若 >7 天，提示「基准数据已过期，建议运行 analyze_all.py」
2. READ: 读取粗纲/世界观（区分: 原始文本 vs 已确认修改）
3. REVIEW: 填满5项检查单
4. SEARCH: >=2次 WebSearch + >=1次 WebFetch（叙事域 >=2+1 足够）
   - WebFetch超时/失败: 降级为WebSearch snippet替代，标记 [降级引用]，不阻塞轮次
5. QUOTE: 从外部来源引用至少 1 条原句（WebFetch失败则用WebSearch snippet，标 [降级引用]）
6. SAMPLE: 从粗纲中引用至少 3 处具体事件/台词/章节号
7. SAMPLE-REF: 将当前粗纲指标与基准库对比，产出 [REF] 偏离标记
8. OUTPUT: [Round N] 格式（见上方模板）
9. BATCH-CHECK:
   a. 批次末(R2/R7/R8)暂停 → 展示本批汇总
   b. 等作者反馈 → 提取决策 → 更新修正确认清单
   c. 作者确认后进入下一批
10. zero: C+H+M>0→zero=0; C+H+M=0→zero+=1
   ⚠️ zero仅在R8生效（R1-R7必须逐轮执行，无视zero值）
11. EXIT: N>=8→[截断]; R8时zero>=2→[收敛]
```

## 修正确认清单机制（防 stale data）

每批暂停后，AI将作者确认的修改记录到清单。下一批分析时假设已确认修改已生效。
清单跨批次持久化在当前对话上下文中（无需额外文件）。

```
修正确认清单格式:
| 来源轮次 | 问题 | 作者决定 | 生效方式 |
|:---:|------|------|------|
| R1 | 开篇冲击力弱 | 改为"啊！"版 | 审查用新版开篇 |
| R2 | 台词逻辑错误 | 改为"不忘记旧账" | 审查用新版台词 |
```

## 暂停超时机制

每批暂停后72h无作者回复：AI自动修复C+H，续跑下一批。

## 护栏规则（防偷懒 v6.1 + Context Isolation）

<!-- ⚠️ 本表与 loop-evolution/SKILL.md、skill-evolution/SKILL.md、chapter-evolution/SKILL.md、exhaustive-search/SKILL.md 护栏规则保持同步。
     修改任一处时必须同步修改另四处（project.mdc rule 32）。 -->

| 违规 | 后果 | 恢复指令 |
|------|------|------|
| 5项全✅但无依据(R3-R7)或定性描述<2句(R1-R2) | 本轮无效 | 补依据后重做 |
| 无外部引用原句 | 本轮无效 | 补搜索+引用后重做 |
| 取样 < 3 处 | 本轮无效 | 补取样后重做 |
| R8之前宣布收敛 | 无效 | 继续下一轮，至少跑满8轮 |
| 搜索词与最近3轮完全相同 | 本轮无效 | 换搜索词后重做 |
| 外部引用为自创 | 本轮无效 | 真实搜索后重做 |
| 跳过搜索直接 found:0 | 本轮无效 | 执行搜索后重做 |
| 取样证据≥3条全旧 | 本轮无效(允许1-2条重复) | 补新取样后重做 |
| 输出含"跳过/同上/前轮已验证" | 本轮无效,重做 | 从零重做 |
| 检查单≥4项引用与前轮相同 | 本轮无效 | 扩大取样范围后重做 |

**Context Isolation**: 每轮不得引用前轮检查结果；从零重读；新取样≥2处与前轮不同(R5-R7:≥1处)
