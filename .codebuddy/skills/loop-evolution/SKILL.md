# 通用穷举进化 v2.1 — 自适应角色系统

**定位**: 用户一句话说"我要进化什么" → AI 先判断任务需要哪些搜索视角 → 固定核心角色 + 动态扩展角色 → 穷举搜索 → 循环产出最优方案。

**与其他 Skill 的关系**: `exhaustive-search`(代码域)/`skill-evolution`(Skill设计域)/`rough-outline-evolution`(粗纲域)/`chapter-evolution`(细纲域)/`loop-evolution` = 外部知识驱动系统优化（不可互相替代）

## 核心设计：角色不由协议写死，由任务决定

### 固定核心角色（每任务必经，4个）

| 角色 | 搜索域 | 说明 |
|------|------|------|
| **学术论文搜索员** | arxiv/ACL/顶会 | 任何系统优化都需要学术前沿参照 |
| **开源项目搜索员** | GitHub/GitLab | 工业界如何工程化实现同类方案 |
| **方法论博文搜索员** | 知乎/CSDN/Medium | 一线实践者的经验教训 |
| **整合排序员** | 前面所有轮次 | 汇总、去重、排优先级 |

### 动态角色库（按任务按需加载）

根据任务特征，从以下角色库中选取 2-4 个加入轮换：

| 动态角色 | 适用场景 | 搜索域 |
|------|------|------|
| **商业工具对比** | 任务涉及工具链/平台/产品 | SaaS/APP/竞品分析 |
| **数据集基准搜索** | 任务涉及数据/评测/指标 | HuggingFace/Kaggle/benchmark |
| **反模式教训搜索** | 任何复杂系统（强烈建议） | "为什么失败/废弃"的教训 |
| **交叉验证员** | 多来源发现存在矛盾时（强烈建议） | 消解矛盾、提取共识 |
| **安全审计视角** | 涉及安全/权限/用户数据的系统 | 安全漏洞/合规风险 |
| **性能优化视角** | 涉及计算/存储/延迟的系统 | 算法效率/资源优化 |
| **用户体验视角** | 涉及交互/流程/界面的系统 | 可用性/认知负荷 |
| **跨领域迁移视角** | 常规搜索乏善可陈时 | 完全不同领域的可迁移方法 |

## Phase 0：任务分析（第一件事，先于所有轮次）

进入搜索前，必须先回答四个问题：

```
① 任务类型判断: 目标系统属于哪个领域？（代码架构 / 数据分析 / 写作工具 / AI模型 / ...）
② 知识覆盖检查: 近 10 轮对话中是否已充分搜索该领域的外部知识？
   YES → 启用「快速模式」→ 每轮只需要 1-2 次 WebSearch（补充缺口而非全面搜索）
   NO  → 标准模式 → 每轮 >=3 次 WebSearch
③ 角色选取: 固定4核心 + 从动态库中选取 ___（列出选中的2-4个动态角色及选取理由）
④ 轮次规划: 快速模式：核心3轮(论文/博文/排序) + 动态0-2轮；
   标准模式：核心4轮 + 动态2-4轮
```

**快速模式适用条件**（满足任意一条即启用）：
- 同一会话中已执行过 >=2 次 WebSearch 覆盖该领域
- 当前对话包含该领域的参考文档/方法论（如马良写作、笔灵拆书）
- 用户明确指出"基于之前搜索的结果"或"不用重新搜索"

**快速模式搜索要求**：
- 每轮 1-2 次 WebSearch（仅搜索新缺口，不重复已知领域）
- 优先复用已有 WebSearch 结果 + 引用上下文中的已有外部知识
- 标注 `[快速·已有知识引用]` 或 `[快速·缺口搜索]`

**动态角色选取原则**: 不是越多越好。选"这个任务绝对需要但还没覆盖"的视角。宁可先少选、发现缺口再加。

## 执行协议

### 执行流程

```
PHASE 0: 任务分析 → 确定模式(标准/快速) + 角色清单 + 轮次总数
PHASE 1: 核心角色轮换（标准: R1-R4 论文→项目→博文→排序 | 快速: R1-R3 论文→博文→排序）
PHASE 2: 动态角色轮换（R5-Rn 或 R4-Rn，按选取顺序执行）
PHASE 2.5: 扩展检查 — 每完成一个动态角色后，审视: "还需要加视角吗？"
```

### Round N 执行步骤

```
0. ROLE-LOAD: 读取 roles/{name}.md
1. SEARCH:
   标准模式: >=3次WebSearch，搜索词体现角色视角 + 目标系统关键词
   快速模式: 1-2次WebSearch（仅搜索新缺口）。已有知识从上下文中引用，标 [快速·已有知识引用]
   - 搜索词不得与最近3轮完全相同
   - WebFetch超时/失败: 降级为WebSearch snippet，标记 [降级引用]
2. EXTRACT: 从搜索结果(或已有上下文)中提取 >=2 条关键发现
3. QUOTE: 引用 >=1 条外部来源原句（快速模式下可从已有搜索/方法论文档中引用）
4. MAP: 每条发现 → 1条"如何应用到目标系统"的具体建议
5. EXPAND-CHECK: 每完成一个动态角色后(Phase 2)，问自己:
   "还有没有这个任务需要的、但还没被搜索的视角？"
   如有 → 从动态库中选取新增角色 → 加入轮换
   如无 → 继续
6. OUTPUT: [Round N] 格式（快速模式下标注 [快速模式]）
7. zero计数: 本轮可操作建议=0 → zero+=1 | >0 → zero=0
8. EXIT: zero>=2 且 N>=min_rounds→[收敛]; N>=12且zero<2→[截断]
```

**min_rounds** = 4(核心) + len(动态角色) + len(扩展新增角色)

**执行模式**: 全自动（跑到底，不暂停）

## 每轮输出格式 (MANDATORY)

```
[Round N] role:角色名（搜索域）

搜索词:
  - "搜询词1"
  - "搜询词2"
  - "搜询词3"

关键发现:
  ① [论文/项目/工具/方法论名称] — [核心 insight，1-2句]
  ② [...]

外部引用: "[原句]" — 来源: [名称], [年份] [降级引用]

┌─ 映射到目标系统 ─────────────────────────────┐
│ 建议1: [具体可操作方案]                          │
│ 建议2: [...]                                    │
└────────────────────────────────────────────────┘

[Phase 2仅] 扩展检查: 需加视角? [是→新增角色:XX] / [否→继续]
产出: N条可操作建议 | zero:N → continue/converge
```

**没有 source/insight/map 三项 = 本轮无效, 不计数**

## 护栏规则

| 违规 | 后果 | 恢复指令 |
|------|------|------|
| 跳过 Phase 0 任务分析直接搜索 | 无效 | 返回 Phase 0 分析 |
| 无外部引用原句 | 本轮无效 | 补搜索+引用后重做 |
| 搜索词与最近3轮完全相同 | 本轮无效 | 换搜索词后重做 |
| 外部引用为自创而非搜索结果 | 本轮无效 | 真实搜索后重做 |
| 跳过搜索直接输出 | 本轮无效 | 执行搜索后重做 |
| 产出0条可操作建议 | 本轮无效 | 重新映射后重做 |
| 收敛前宣布完成 | 无效 | 继续下一轮，至少跑满 min_rounds |
| 输出含"跳过/同上/前轮已验证" | 本轮无效,重做 | 从零重做 |
| 快速模式下跳过 Phase 0 知识覆盖检查 | 无效 | 返回 Phase 0 确认模式 |
| 快速模式下仍执行 >=3 次冗余搜索 | 浪费 | 缩减至1-2次，优先引用已有知识 |
| 知识覆盖不足却启用快速模式 | 输出不完整 | 降级回标准模式重新搜索 |

## 执行纪律 — ⚠️ 最高优先级

### 诚实自查清单（每轮输出前必查）

| # | 标准 | 违规示例 |
|---|------|---------|
| ① | Phase 0 是否已输出？ | 直接搜了没声明模式 |
| ② | zero 计数是否诚实？ | 找到方向性确认却标 zero=1 |
| ③ | 搜索词是否与前三轮不同？ | R1搜了"essay scoring" R4又搜同样的 |
| ④ | 引用来自搜索结果还是自创？ | "如XX论文所述"但根本没搜到 |
| ⑤ | 按角色视角搜的还是随便搜的？ | 开源角色却搜学术关键词 |
| ⑥ | 是否跳过轮次？合并轮次？一轮输出多轮结果？ | R4直接复制R3发现标两轮 |
| ⑦ | 收敛是真的枯竭还是偷懒？ | zero>=2但R5/R6根本没搜就标zero |
| ⑧ | 动态角色是否按需扩展？ | Phase 2搜了两轮无发现但没加新角色 |

**违反任意一条 → 本轮无效，从违规处重做。** 学习记录：2026-06-08 穷举对比引擎时，AI未输出Phase 0就搜了R1，R2有发现却标zero=1凑收敛，R4排序员占独立轮次不合规。

## 终止条件

zero>=2 且 N>=min_rounds → [收敛] | N>=12且zero<2 → [截断]

**收敛机制**: 连续2轮搜索无法产出新的可操作建议 → 外部知识池已枯竭 → 收敛。
**收敛前必查**: 所有角色视角都搜过了？搜索词没有重复？zero确实是搜索无果不是偷懒？
