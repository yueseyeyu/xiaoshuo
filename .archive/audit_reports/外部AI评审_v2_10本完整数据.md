### 1.1 章节解析器 (4 种格式)

```python
def extract_chapters(filepath):
    """Supports: 第X章 / 序章+章X / 数字标题 / 纯数字"""
    text = Path(filepath).read_text(encoding=enc)
    cn_nums = r"[一二三四五六七八九十百千零\d]+"

    # 优先尝试「第X章」
    pattern1 = r"(第" + cn_nums + r"章\s*[^\n]*)"
    parts = re.split(pattern1, text)
    if len(parts) >= 5: return _build_chapters(parts)

    # 混合格式 (序章 + 章X，如狩魔手记)
    pattern2 = r"(序章\s*[^\n]*|章" + cn_nums + r"\s*[^\n]*)"
    parts = re.split(pattern2, text)
    if len(parts) >= 5: return _build_chapters(parts)

    # 数字标题 (如限制级末日症候: "1 厕所怪谈")
    pattern3 = r"(^[ \t]*\d{1,4}[ \t]+[^\d][^\n]*)"
    parts = re.split(pattern3, text, flags=re.MULTILINE)
    if len(parts) >= 5: return _build_chapters(parts)

    # 纯数字行 (fallback)
    pattern4 = r"(^[ \t]*\d{1,4}[ \t]*$)"
    parts = re.split(pattern4, text, flags=re.MULTILINE)
    if len(parts) >= 5: return _build_chapters(parts)
    return []
```

### 2.2 百分位评分 — 秩排序 (v4)

```python
def percentile_score(value, pool, metric):
    sorted_vals = pool[metric]["_sorted"]    # 10 本书的排序值
    rank = sum(1 for v in sorted_vals if v <= value)  # 胜过几本
    n = len(sorted_vals)
    return round((rank - 1) / (n - 1) * 100)          # 0-100 秩百分位
```

## 四、评审问题清单

### A. 拆书逻辑可信度

1. **关键词覆盖率**: 现在有 9 类爽点 + 5 类冲突，但仍是纯正则匹配。对于「月光落在她肩上」这种零关键词但极高情感浓度的句子，覆盖率是否仍然严重不足？能否估算当前覆盖率相对于 LLM 判断的比例？

2. **冲突检测的有效性**: 5 类冲突中，新增的心理/道德/环境/社会关键词是否提供了足够的区分度？它们在 10 本书中的实际匹配率分布是怎样的？

3. **爽点强度公式的权重**: 公式中的 1.5/1.0/8/0.5/0.3/1.5 权重全部是经验值。现在有 LLM 内置抽查数据(r=0.3-0.67)，能否用这些数据对权重做一次线性回归拟合？

4. **LLM 内置验证 vs 独立校准**: 内置验证 r=0.3-0.67 (抽查 prompt 只让 LLM 修正已有标签)，独立校准 r=0.16 (让 LLM 从零打分)。两个 r 值差异巨大→说明什么？哪个更可信？

5. **章节解析器的 4 种格式**: 是否还有遗漏？中文网文常见的「卷X 第Y章」「Act.X」「Episode X」等格式是否覆盖？

### B. 商业评分可信度

6. **n=10 的秩排序稳定性**: n 从 5→10，秩排序的分辨率从 5 档→10 档。但排名完全取决于同池中其他书的表现。如果池子本身分布有问题(比如缺了某一档的书)，会怎么影响评分？

7. **打脸频率的子类型分化**: 打脸/智斗/羁绊三档的区间(0.25-0.5/0.05-0.15/0.05-0.1)有没有数据支撑？直接减少了哪些书的打脸扣分？

8. **反转/悬念/大爽 P50 锁死问题是否缓解**: 10 本池中这三维的区分度现在如何？秩排序下它们的实际方差有多大？

9. **权重 50/30/20**: 扰动分析(±10%)下评级稳定的比例是多少？如果跳级率 >20%，评分结论对用户就不可靠。

10. **完读率预估的验证**: Survival 分段完读率与商业评分的关系有验证吗？比如限制级末日症候完读率 93-97% 但商业分只有 52——这合理吗？

### C. 对《末日模拟器》的适用性

11. **智斗流对标**: 池中最接近的参照物是黑暗血时代(48分)和限制级末日症候(52分)。但它们的钩子密度(0.84/1.13)远低于题材均值(1.99)。《末日模拟器》的前 3 章设计了三层钩子→如果它的钩子密度不低，当前的智斗流参照是否偏悲观？

12. **羁绊爽点的量化**: 10 本书中羁绊成为「主流爽点」——这是 v4 关键词的产物还是真实反映？羁绊关键词匹配到的内容，有多大比例属于「真正的情感羁绊」vs「泛化的社交互动」？
