## 四、两份评审分别提出的独特建议

### Kimi 独有建议

1. **已知精品名单保护**: 18本名单中的书如果因格式问题被过滤，不应硬降级——这些是人工确认的高价值数据
2. **`rule_translator` 详细范式**: 给出了"指标→规则→动作"的具体转换示例表
3. **配置拆分方案**: `config.yaml` + `weights.yaml` + `patterns/` 三级
4. **评分版本标记**: `score_version` 字段让下游知道置信度
5. **7天 QUARANTINE 自动降级**: 无人工干预则自动→FAIL

### DeepSeek 独有建议

1. **函数型主成分分析（fPCA）**: 提取跨书共同冲突模式
2. **冷启动池**: 人工预设10-20本经典爆款统计基线，新书直接算Z-score
3. **`GuideInjector` 中间件**: 读取 guidance.json → Agent prompt 注入规则
4. **主题模型(LDA)**: 分析本书主题与精品池差异度 → 创新信号
5. **动态时间规整(DTW)**: 比较冲突升级曲线相似度 → 套路检测
