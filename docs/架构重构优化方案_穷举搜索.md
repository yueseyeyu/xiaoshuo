| 中间产物 | ✗ | DB | DB | `data/processed/` | ✅ 已正确 |

## 三、现有架构诊断（无角色视角）

### 3.1 结构性问题

| # | 问题 | 严重度 | 影响 |
|---|------|--------|------|
| 1 | **`analysis/outputs/` 与 `outputs/reports/` 两套输出路径** | 🔴 | 代码写入 vs 人工归档路径不一致，新开发者困惑 |
| 2 | **`analysis/` 和 `agents/` 在根目录** | 🔴 | 不符合 Python 生态标准 (src-layout)，无法 pip install -e . |
| 3 | **`prompts/` 目录职责不清** | 🟡 | 同时存 Marvis 审查工作流 + 项目指令模板，混淆 |
| 4 | **`docs/` 含 2 个副本文件 + 1 个误放系统 prompt** | 🟡 | README 副本、marvis_system_prompt 不应在 docs/ |
| 5 | **`assets/` 与 `data/processed/` 边界模糊** | 🟡 | assets 是作者创作资产(手写)，data/processed 是程序产物，但命名不直观 |
| 6 | **缺 `notebooks/` 探索目录** | 🟡 | 所有探索性分析应隔离，不污染生产代码 |
| 7 | **缺 `Makefile` 或任务运行器** | 🟢 | 常用命令散落在 README / scripts / CODEBUDDY.md 三处 |
| 8 | **`.env.example` 缺失** | 🟢 | DeepSeek API key 硬编码在 models.json 中 |
| 9 | **`state.json` 在根目录** | 🟢 | 运行时状态文件应归入 data/ 或专用目录 |
| 10 | **`SKILL.md` (根) 与 `.codebuddy/skills/` 命名冲突** | 🟢 | 同名不同义，建议 SKILL.md → AI_PROTOCOL.md |

### 3.2 代码层问题

| # | 问题 | 来源 |
|---|------|------|
| A | `analysis/` 19 个 .py 无 `__init__.py` 包声明 | 非标准 import |
| B | `agents/` 多个模块导入 analysis/，形成隐式回路 | 循环依赖风险 |
| C | `config.yaml` 精品书单 6 本重复 | 数据冗余 |
| D | 4 个文件版本号不一致 (v7.1/v7.3/v7.4) | 维护失控 |

## 四、行业最佳实践提炼

### 4.1 Python 项目结构黄金法则（2025-2026 共识）

```
project/
├── pyproject.toml          # 唯一构建/依赖/元数据入口
├── .env.example            # 环境变量模板（敏感值不提交）
├── Makefile                # 常用命令快捷方式
├── README.md               # 项目门面（<150 行）
├── src/                    # 源代码包（Poetry 2.1+ 默认）
│   └── project_name/
│       ├── __init__.py
│       ├── config.py       # 配置类（或 config/ 目录）
│       ├── data/           # 数据加载/处理
│       ├── models/         # 模型定义/训练
│       └── utils/          # 工具函数
├── config/                 # 非 Python 配置文件
├── data/                   # 数据 (gitignored except raw/ small samples)
│   ├── raw/                # 原始不可变
│   ├── processed/          # 预处理后
│   └── interim/            # 中间临时
├── notebooks/              # 探索 Notebooks
├── tests/                  # 测试（镜像 src/ 结构）
├── scripts/                # 一次性脚本
├── outputs/                # 生成产物（报告/图表/模型）
└── docs/                   # 文档
```

### 4.2 src-layout 四大收益

| 收益 | 说明 |
|------|------|
| 防误导入 | 开发时不会意外导入系统已安装的旧版本 |
| pip install -e . | 可编辑安装后 import 路径与生产一致 |
| IDE 补全 | 100% 精准，不受 PYTHONPATH 污染 |
| 构建一致性 | 开发/测试/生产行为完全一致 |

### 4.3 网文 AI 创作系统特有模式

对比 NovelBuilder、aiNovel、ProseEngine、本项目的共性：

```
每个网文 AI 系统至少包含：
├── 创作管线     (workflow / pipeline / analysis)     ← 本项目 analysis/
├── AI 行为层    (agents / orchestrator)              ← 本项目 agents/
├── 知识/记忆层  (graph / vector / semantic)          ← 本项目 semantic_memory (P1)
├── 质量关卡     (quality_gate / validation)          ← 本项目 quality_gate
├── 知识注入     (hard rules + soft rules)            ← 本项目 SKILL.md + project.mdc
└── 创作资产     (canon / outline / chapters / voice)  ← 本项目 assets/
```

本项目在这六个维度**全部覆盖**，属同类项目中最完整者之一。

## 五、优化方案

### 方案总览：三阶段递进

```
Phase 1: 无破坏性清理 (P0, 今天可做)

    → analysis/ agents/ → src/xiaoshuo/ + pyproject.toml 升级
```
